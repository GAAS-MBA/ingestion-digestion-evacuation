# ストレージ構成

ドキュメントをストレージ化する際のフォルダ構成。ユニバーサル（産業横断）と産業特性に依存したものを分類する。

---

## 構成

```
storage/
├── universal/                    # ユニバーサル（産業横断）
│   ├── governance/               # ガバナンス・コンプライアンス
│   ├── safety/                   # 安全の基本
│   ├── consumer/                 # 消費者問題・賠償
│   └── lessons-learned/          # 汎用の教訓
│
├── industry/                     # 産業特性に依存
│   ├── retail/                   # 小売
│   ├── manufacturing/            # 製造
│   └── construction/             # 建設
│
└── meta/                         # メタ（構成・運用）
```

---

## 分類の原則

| カテゴリ | 内容 |
|----------|------|
| **universal** | 産業を問わず適用可能なテンプレート、フレームワーク、プロトコル |
| **industry** | 小売・製造・建設など、業種固有の事故種別、規制、フレームワーク |
| **meta** |  taxonomy、保持期間、アクセス制御など、ストレージ自体の運用 |

---

## 運用

- **universal** を土台とし、**industry** で補完する
- 新産業追加時は `industry/{new}/` を追加。**universal** は共通利用
- **meta** で保持期間・アクセス制御・分類体系を管理
