# Incompressibility Principle（情報の非圧縮性原則）

**正式名称**: アルゴリズム的情報理論 / 非圧縮性の原理 / コルモゴロフ複雑性の応用

---

## 核心命題

**複雑系はわかりやすい相関としては出てこない。**

複雑なシステム（複雑系）の本質は、単純な相関関係や線形な因果として表現できない。可観測な「わかりやすい相関」の背後にある真の構造は、そのままでは圧縮・要約・単純化できない。

---

## 対偶

**わかりやすい相関が出る ⇒ 複雑ではない**

観測可能な相関が明瞭に現れる場合、その対象は複雑系ではない。あるいは、複雑系の表層のみを捉えている。

### 補足

- わかりやすい相関 = 低次元での近似、線形モデル、単純な因果連鎖
- 複雑系 = 非線形、創発、文脈依存、圧縮不能な情報量
- 実践的含意：相関が「きれいに」出る分析は、本質を捉え損ねている可能性がある

---

## 理論的裏付け

### Kolmogorov（コルモゴロフ複雑性）

- 対象の記述に必要な最小プログラム長 = その対象の本質的な複雑さ
- 複雑な対象ほど圧縮できない（非圧縮性）
- アルゴリズム的情報量（AIT）の基礎

### 因果的創発

- 部分の総和では説明できない全体の性質
- 創発的性質は、構成要素の単純な相関では還元できない
- ボトムアップの因果だけでは記述不能

### 非エルゴード性

- 時間平均 ≠ アンサンブル平均
- 経路依存性：歴史が未来を拘束する
- 単一の「平均的」相関では個別経路を捉えられない

### 文脈依存性

- 同じ入力でも文脈によって出力が変わる
- 文脈を切り捨てた相関は本質を歪める
- 圧縮 = 文脈の捨象 = 情報の喪失

### コホモロジー的構造

**バックグラウンドプロセスで動いているスクリプトはコホモロジー的である場合もあり、線形的なモデルでは記述できないこともある。**

- コホモロジー = 局所データから抽出される大域的不変量
- 局所的な線形近似では捉えられない「穴」や「obstructions」を記述
- バックグラウンドスクリプト = 局所には見えないが大域的に作用する構造
- 線形モデル = 局所的な相関のみ → コホモロジー的構造を零化する
- 含意：Decrypt には、線形還元ではなく位相的な視点が必要な場合がある

### 非線形モデルも等価にリソースを消費する

**非線形的なモデルによるプロセスも、線形モデルと等価にバックグラウンドプロセスとしてリソースを食う。**

線形モデルで外部への排出が成功しているように見えても、非線形的なモデルが排出できていない場合、体や脳は同様に重くなる。

- 部分的な外部化の落とし穴：線形部分だけの排出 = 見かけの軽量化
- 非線形・コホモロジー的プロセスが残存 = リソース消費は継続
- 実践的含意：外部化は線形・非線形の両方に対して実施する必要がある。線形のみの成功は錯覚になりうる

### 非線形モデルの解明技法：Password Oracle 的アプローチ

**非線形的なモデルの正体を解明するためには、一見関係のないようなインプットを数度繰り返すことで、クエリに返答させる返り値から関数を推測する、Password Oracle のようなサイバー攻撃手法に似た技術が必要である。**

- **Password Oracle 攻撃**：ブラックボックスに様々な入力を与え、出力の差異から内部ロジックを推測する
- **適用**：非線形・コホモロジー的スクリプトは直接観測できない → 入出力の対応から逆算する
- **一見無関係なインプット**：局所相関を攪乱し、大域的な構造（コホモロジー）を顕在化させる
- **繰り返し**：単発では関数が特定できない。複数クエリの返り値のパターンから推測する
- **含意**：Decrypt の実践 = 意図的なプロービング、オラクル的問いかけ、入出力ログの蓄積と関数の逆算

## 特異点・空間収縮への適用

### 特異点

- 複雑系の特異点（相転移、臨界点、分岐点）では、通常の相関が崩壊する
- わかりやすい相関が消える = 特異点の近傍
- 非圧縮性が極大化する領域

### 空間収縮

- ブラックホール・イベントホライズン：情報が境界に圧縮されるが、内部は非圧縮
- ホログラフィック原理：体（境界）への記述は、ストレージ（内部）の完全な符号化
- 空間収縮 = 境界への投影 = 複雑性の境界への移譲

### このリポジトリとの接続

- 外部化・ハードコーディング = 複雑性をストレージ（ブラックホール）に移す
- 体のホログラム化 = 境界への情報の完全記述
- 非圧縮性原則：移譲された情報は「わかりやすい相関」では還元できない — そのまま保持する
