# マテリアライゼーションとストレージ

ハードコーディングされていない探索活動の反復と、ストレージ化による抑制。

---

## 核心命題

**人間はハードコーディングできていない探索活動については、必ずマテリアライゼーションする性質がある。**

- **探索** = 試行・実験・冒険
- **マテリアライゼーション** = その探索が実際の行為・現象として具現化すること
- **ハードコーディングされていない** = ストレージに蓄積されていない = 学習が外部化されていない

### 含意

| 状態 | 結果 |
|------|------|
| **ストレージ化されている** | 同じ探索は起きにくい（履歴が参照される） |
| **ストレージ化されていない** | 同じ探索が繰り返しマテリアライゼーションする |

---

## ケーススタディ

### Case 1：会計不正と Museology

**ストレージ化の例**：NYSE、NASDAQ における会計不正の歴史が蓄積され、Museology（事例の展示・保管・教育への組み込み）になっている。

- 不正事例がデータベース化・分析・ルール化されている
- その土台があるため、同じ会計不正は起きにくくなる
- スチュアードシップ・ガバナンスルールが参照可能な形で存在する

**非ストレージ化の例**：日本、インドネシアなどの市場では、NYSE・NASDAQ に多くの事例があっても、データが伝達されない。

- データベース化されていない、あるいは共有経路が弱い
- スチュアードシップやガバナンスルールが一般適用されていない
- 同じ探索（会計不正）が繰り返しマテリアライゼーションするリスクが高い

### Case 2：核戦争による文明リセットとスヴァールバル

**ノルウェー・スヴァールバル**：世界種子貯蔵庫（Global Seed Vault）や文明のアーカイブが置かれている。

**ストレージがない場合**：
- 核戦争 → 文明リセット → 知識・教訓の喪失
- 喪失した状態で再び「探索」が始まる
- 核兵器開発・核戦争という同じ探索が、再びマテリアライゼーションする
- 同じサイクルが繰り返される

**ストレージがある場合（スヴァールバル）**：
- 文明のアーカイブが外部に保存される
- リセット後も、知識・教訓が参照可能な形で残る
- 「核戦争という探索」の履歴がストレージ化されている
- 同じ探索を繰り返す必要性が下がる、あるいは繰り返しても回復の土台が残る

**一般化**：文明アーカイブがなければ、リセット後に同じ探索（核戦争など）が再び具現化しただろう。ストレージの有無が、反復の有無を決める。

---

## このリポジトリとの接続

- **外部化・ハードコーディング** = 探索の履歴をストレージに移す = マテリアライゼーションの抑制
- **非ストレージ化** = 探索が繰り返し具現化する = 同じ失敗・同じパターンの再生
- **データの伝達** = ストレージが地理的・制度的に共有されるか。共有されないと、他市場・他文明の教訓は届かない
- **Museology** = 歴史を「展示・保管・参照可能な形」でストレージ化すること。会計不正の文脈では、事例の蓄積・分析・教育への組み込み
- **文明アーカイブ** = Museology の極端な形。文明全体の履歴を、物理的に安全な場所に保管する
