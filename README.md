# ingestion-digestion-evacuation

**v0.0.1**

体のパフォーマンスを最適化するための知恵を実装するリポジトリ。

---

## 目的

人格・性格・能力を作り上げている一連のアルゴリズムの組み合わせを **Decrypt（解読）** し、**再現性** を持たせること。

```
暗号化された自己（自動スクリプトの集合）
        ↓ Decrypt
明示されたアルゴリズム群
        ↓ 外部化・ハードコーディング
再現可能なデジタルツイン
```

---

## 背景となる一般則

情報というエントロピーの**摂取・消化・排泄**が適切に実施されていない場合、体や脳というコンピューティングは：

- 処理が重くなる
- キューが溜まる
- 動きが遅くなる

これを止めるためには、マクスウェルの悪魔的に自動で実施される「**選択**」のバックグラウンドプロセスを「止める」必要がある。

**止める** = 特定のプロセス自体をパッケージングして外部アタッチメントとしてハードコーディングする。

これは人間が厚い皮や体毛、爪を減らし体をセンサー・プロセッサとして進化させた適応の歴史と類似する。

## 機能

### 1. 自動スクリプトの客観的特定

自分が無意識に・自動的に行っているスクリプト（判断、反応、習慣）を客観的に特定する。

### 2. デジタルツインとしての外部化

特定したスクリプトを、以下のような手段で外部化する：

| 手段 | ハードコーディング度 | 外部ストレージ化の成功しやすさ |
|------|----------------------|-------------------------------|
| GitHubでコード化 | ◎ 高い | 高い |
| AIにコメントアウト | ◎ 高い | 高い |
| ホームページ | ○ 中〜高 | 中〜高 |
| ブログ | ○ 中 | 中 |
| 書籍 | △ 中 | 中 |
| 商品 | △ 中〜低 | 中〜低 |
| 会社 | △ 低 | 低 |

**ハードコーディングできる手法が多いほど、外部ストレージ化に成功しやすい。**

### 3. 再現性の確保

- 曖昧さを排除する
- 同じ入力で同じ出力を再現できるようにする
- 判断の負荷を本体から外す
- バージョン管理・進化を可能にする
